---
title: useMainDevice
description: Read the GPUDevice from GPUDeviceProvider context.
---

`useMainDevice` returns `{ adapter, device }` from the nearest [`GPUDeviceProvider`](/api/gpu-device-provider). Use it in child components so only the provider requests the device once.

## Usage

Wrap your app (or scene subtree) in `GPUDeviceProvider`, then read the shared device in any descendant:

```tsx
import { useEffect } from "react";
import { PixelRatio } from "react-native";
import {
  Canvas,
  GPUDeviceProvider,
  useCanvasRef,
  useMainDevice,
} from "react-native-webgpu";

function Scene() {
  const ref = useCanvasRef();
  const { device } = useMainDevice();

  useEffect(() => {
    if (!device) return;

    const context = ref.current!.getContext("webgpu")!;
    const canvas = context.canvas as HTMLCanvasElement;
    const format = navigator.gpu.getPreferredCanvasFormat();

    canvas.width = canvas.clientWidth * PixelRatio.get();
    canvas.height = canvas.clientHeight * PixelRatio.get();
    context.configure({ device, format, alphaMode: "premultiplied" });

    // Shared device — safe to pass to Three.js, overlays, debug UI, etc.
  }, [device, ref]);

  return <Canvas ref={ref} style={{ flex: 1 }} />;
}

function DebugOverlay() {
  const { adapter } = useMainDevice();
  // Read adapter info, feature list, etc.
  return null;
}

export function App() {
  return (
    <GPUDeviceProvider
      deviceDescriptor={{
        requiredFeatures: ["rnwebgpu/native-texture" as GPUFeatureName],
      }}
    >
      <Scene />
      <DebugOverlay />
    </GPUDeviceProvider>
  );
}
```

<Callout type="warn" title="Must be inside provider">
Throws `No DeviceContext found.` if called outside `GPUDeviceProvider`. The provider renders `null` until the device is ready, so children only mount once `device` is non-null.
</Callout>
