---
title: GPUOffscreenCanvas
description: Offscreen render target with CPU readback via getImageData().
---

`GPUOffscreenCanvas` creates an offscreen WebGPU surface for headless rendering and testing. There is no swapchain — skip `present()` and read pixels back with `getImageData()`.

## Constructor

```tsx
new GPUOffscreenCanvas(width: number, height: number)
```

Dimensions are fixed at construction (unlike on-screen [`Canvas`](/api/canvas), which follows layout).

## Methods

| Method | Description |
| ------ | ----------- |
| `getContext("webgpu")` | Returns a `GPUCanvasContext` for offscreen rendering |
| `getImageData()` | Copies the rendered texture to CPU and returns `{ data, width, height, format }` |

## Usage

Render a solid color offscreen and read pixels back for assertions:

```tsx
import { GPUOffscreenCanvas } from "react-native-webgpu";

async function renderRedPixel(device: GPUDevice) {
  const canvas = new GPUOffscreenCanvas(1, 1);
  const context = canvas.getContext("webgpu")!;

  context.configure({ device, format: "rgba8unorm" });

  const fullscreenVertWGSL = `
@vertex fn main(@builtin(vertex_index) i: u32) -> @builtin(position) vec4f {
  var pos = array<vec2f, 3>(vec2(-1, -1), vec2(3, -1), vec2(-1, 3));
  return vec4f(pos[i], 0.0, 1.0);
}`;

  const pipeline = device.createRenderPipeline({
    layout: "auto",
    vertex: {
      module: device.createShaderModule({ code: fullscreenVertWGSL }),
      entryPoint: "main",
    },
    fragment: {
      module: device.createShaderModule({
        code: `@fragment fn main() -> @location(0) vec4f {
          return vec4(1.0, 0.0, 0.0, 1.0);
        }`,
      }),
      entryPoint: "main",
      targets: [{ format: "rgba8unorm" }],
    },
    primitive: { topology: "triangle-list" },
  });

  const encoder = device.createCommandEncoder();
  const pass = encoder.beginRenderPass({
    colorAttachments: [{
      view: context.getCurrentTexture().createView(),
      clearValue: [0, 0, 0, 1],
      loadOp: "clear",
      storeOp: "store",
    }],
  });
  pass.setPipeline(pipeline);
  pass.draw(3);
  pass.end();

  device.queue.submit([encoder.finish()]);
  await device.queue.onSubmittedWorkDone();

  const { data, width, height, format } = await canvas.getImageData();
  // data is RGBA bytes — compare against expected values in tests
  return { data, width, height, format };
}
```

Typical test flow:

1. Create `GPUOffscreenCanvas` at fixed dimensions.
2. Render a deterministic scene (solid colors, known geometry).
3. `await canvas.getImageData()` → compare checksums or per-pixel diff.
4. Assert pass/fail.

## When to use offscreen vs Canvas

| | `<Canvas>` | `GPUOffscreenCanvas` |
| --- | ---------- | --------------------- |
| Visible output | Yes | No |
| `present()` | Required | Not used |
| Size | From layout | Fixed at construction |
| Use case | Apps, games, AR | Tests, batch processing |
