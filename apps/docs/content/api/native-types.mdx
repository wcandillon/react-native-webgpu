TypeScript definitions for native media interop. Import values from `react-native-webgpu` or read them from `RNWebGPU` helpers — the types below describe the shapes you work with.

## Video

### NativeVideoFrame

Opaque handle to a decoded video or camera frame. Created by `RNWebGPU.createVideoPlayer`, `createTestVideoFrame`, `createVideoFrameFromNativeBuffer`, etc.

```tsx
import type { NativeVideoFrame } from "react-native-webgpu";

let currentFrame: NativeVideoFrame | null = null;

const newFrame = player.copyLatestFrame();
if (newFrame) {
  currentFrame?.release();
  currentFrame = newFrame;

  const texture = device.importExternalTexture({ source: currentFrame });
  // draw… then after queue.submit():
  texture.destroy();
  currentFrame.release();
  currentFrame = null;
}
```

Always call `release()` when done — it returns the native buffer to the pool.

### VideoPlayer

Decoded MP4 player returned by `RNWebGPU.createVideoPlayer`:

```tsx
import type { VideoPlayer } from "react-native-webgpu";

const player: VideoPlayer = RNWebGPU.createVideoPlayer(url, "nv12");
player.play();

const frame = player.copyLatestFrame(); // null if no new frame yet
player.pause();
player.release(); // when unmounting
```

## Shared texture memory

Import native pixel surfaces without copying through the CPU. Requires `"rnwebgpu/native-texture"`.

```tsx
import type { GPUSharedTextureMemoryDescriptor } from "react-native-webgpu";

const memory = device.importSharedTextureMemory({
  handle: frame.handle,
  label: "video-frame",
} satisfies GPUSharedTextureMemoryDescriptor);
```

## Shared fences

Synchronize GPU work across producer and consumer (e.g. camera pipeline → WebGPU render):

```tsx
import type { GPUSharedFenceDescriptor } from "react-native-webgpu";

const fence = device.importSharedFence({
  handle: exportedHandle,
  label: "frame-sync",
} satisfies GPUSharedFenceDescriptor);

fence.wait(0);
// … sample imported texture …
fence.signal(0);
fence.destroy();
```
