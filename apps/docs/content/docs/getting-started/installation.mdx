React Native WebGPU brings the [WebGPU API](https://www.w3.org/TR/webgpu/) to iOS, Android, macOS, and visionOS using [Dawn](https://dawn.googlesource.com/dawn). If you have never used WebGPU before, think of it as a modern GPU API: you write **WGSL shaders**, build **pipelines**, encode **command buffers**, and submit work to a **queue**. React Native WebGPU keeps that model identical to the browser - the main differences are manual frame presentation and native texture import.

## Requirements

| Requirement | Notes |
| ----------- | ----- |
| **React Native ≥ 0.81** | Required |
| **New Architecture** | Legacy architecture is **not** supported |
| **Physical device** | Recommended for camera, native textures, and realistic GPU performance |

## Install the package

```npm
npm i react-native-webgpu
```

<Callout type="info" title="Previously react-native-wgpu">
The package was previously published as `react-native-wgpu`. A shim with that name still re-exports `react-native-webgpu` but logs a deprecation warning in development.
</Callout>

| Package | Version | Required |

For UI-thread rendering with Reanimated:

```npm
npm i react-native-reanimated react-native-worklets
```

See [Reanimated](/docs/integrations/reanimated) for the full integration guide.

## Native setup

Prebuilt Dawn binaries are included for iOS, Android, macOS, and visionOS. After installing the npm package:

```bash
# iOS / macOS / visionOS
cd ios && pod install

# Android - rebuild the app (no extra steps beyond a normal RN build)
```

Restart Metro and rebuild the native app after the first install.

## What you get

Once installed, the global `navigator.gpu` API is available - the same entry point as in Chrome or Safari:

```tsx
const adapter = await navigator.gpu.requestAdapter();
const device = await adapter!.requestDevice();
```

Rendering happens inside a [`Canvas`](/api/canvas) view. The typical loop is:

1. Get a WebGPU context from the canvas ref
2. Configure the context with your device
3. Encode render (or compute) passes
4. `device.queue.submit(...)`
5. **`context.present()`** - required on React Native ([Present](/docs/rendering/present))

Start with [Hello Triangle](/docs/rendering/hello-triangle) once installation is done.

## Troubleshooting

<Callout type="warn" title="Android emulators">
On Android emulators, WebGPU may fall back to a **software adapter**. Rendering is slow and some features (especially native textures) may be unavailable. Use [`warnIfNotHardwareAccelerated`](/api/warn-if-not-hardware-accelerated) to log a warning during development. Prefer physical devices for camera, video, and performance testing.
</Callout>

<Callout type="warn" title="iOS Simulator - Metal API Validation">
When running on the **iOS Simulator**, disable **Metal API Validation** in Xcode or you may hit false-positive errors and crashes that do not occur on device.

**Steps:** Product → Scheme → Edit Scheme → Run → Diagnostics → uncheck **Metal API Validation**.

Metal API Validation is a debug mode that checks every Metal call against the API specification. The simulator's Metal stack is not identical to real hardware, and validation can flag legitimate Dawn/WebGPU usage as invalid. Disabling it for local development is standard practice; keep validation enabled when debugging raw Metal code on device if needed.

Learn more in [Apple's documentation on validating Metal API usage](https://developer.apple.com/documentation/xcode/validating-your-apps-metal-api-usage/).
</Callout>
