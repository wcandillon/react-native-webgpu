---
title: Transparency
description: Render WebGPU content over React Native views.
---

By default, a `Canvas` clears to an opaque background. To composite GPU output **over** React Native views (gradients, images, UI), enable transparency on the canvas and configure alpha correctly.

## Canvas prop

Stack the GPU layer above your RN content:

```tsx
<View style={{ flex: 1 }}>
  <View style={{ flex: 1, backgroundColor: "#3498db" }} />
  <Canvas ref={ref} style={StyleSheet.absoluteFill} transparent />
</View>
```

The `transparent` prop tells the native surface to support alpha compositing.

## Context configuration

Configure the swapchain with premultiplied alpha:

```tsx
context.configure({
  device,
  format: navigator.gpu.getPreferredCanvasFormat(),
  alphaMode: "premultiplied",
});
```

In your render pass, clear with alpha `0` where you want to see through to the RN view below:

```tsx
colorAttachments: [{
  view: textureView,
  clearValue: [0, 0, 0, 0], // RGBA - alpha 0 = transparent
  loadOp: "clear",
  storeOp: "store",
}],
```

Fragment shaders should output premultiplied colors when using this mode (RGB already multiplied by alpha).

<Callout type="warn" title="Android">
On **Android**, `alphaMode` is **ignored** during `context.configure()`. Use the `transparent` prop on `Canvas` - that is what enables transparency on Android. On iOS/macOS, both `transparent` and `alphaMode: "premultiplied"` apply.
</Callout>

## When to use transparency

| Use case | Approach |
| -------- | -------- |
| HUD over RN UI | Transparent canvas + clear alpha 0 in unused regions |
| Full-screen GPU scene | Opaque canvas, no `transparent` prop |
| Video/camera overlay | Often opaque GPU pass with alpha in shader, or transparent canvas for partial overlays |
