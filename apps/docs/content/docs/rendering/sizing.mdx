title: Sizing & Pixel Ratio
description: Scale canvas buffers to match device density and handle resize.

WebGPU does not automatically scale your drawing buffer to the device pixel ratio. You control buffer dimensions explicitly - the same rule as on the web.

## Layout vs buffer size

Two sizes matter:

| Property | Meaning |
| -------- | ------- |
| `canvas.clientWidth` / `clientHeight` | Layout size in **CSS/logical pixels** - updates when the view resizes |
| `canvas.width` / `canvas.height` | **Physical pixel** dimensions of the GPU buffer you draw into |

On a 3× Retina screen, a 400×300 pt view has `clientWidth = 400` but should use `width = 1200` for sharp output.

```tsx
import { PixelRatio } from "react-native";

const ctx = ref.current!.getContext("webgpu")!;
const canvas = ctx.canvas as HTMLCanvasElement;

const dpr = PixelRatio.get();
canvas.width = canvas.clientWidth * dpr;
canvas.height = canvas.clientHeight * dpr;
```

Do this **before** `context.configure()` and whenever the view resizes.

## Handling resize

When the user rotates the device or a parent view animates, `clientWidth` and `clientHeight` update automatically. Your render loop should detect changes and:

1. Recompute `canvas.width` / `canvas.height`
2. Recreate depth textures, MSAA targets, and any size-dependent attachments
3. Update projection matrices (aspect ratio)

```tsx
let lastWidth = 0;
let lastHeight = 0;

const render = () => {
  const w = canvas.clientWidth;
  const h = canvas.clientHeight;

  if (w !== lastWidth || h !== lastHeight) {
    lastWidth = w;
    lastHeight = h;
    canvas.width = w * PixelRatio.get();
    canvas.height = h * PixelRatio.get();
    recreateDepthTexture(canvas.width, canvas.height);
  }

  // encode, submit, present…
};
```

If `clientWidth === 0`, the view is not laid out yet - skip rendering and retry on the next frame.

## Three.js and R3F

Third-party renderers may also set buffer size. When using React Three Fiber, size the native canvas with `PixelRatio.get()` and pass `dpr: 1` to R3F so dimensions are not doubled - see [React Three Fiber](/docs/integrations/react-three-fiber).

<Callout type="info">
Unlike `present()` and native textures, sizing and pixel density work symmetrically with browser WebGPU - no extra RN-specific APIs beyond `PixelRatio.get()`.
</Callout>
