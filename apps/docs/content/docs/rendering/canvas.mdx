The [`Canvas`](/api/canvas) component is a React Native view that owns a GPU surface. Every pixel you draw with WebGPU passes through it - there is no hidden fullscreen context like in a browser tab.

If you are new to WebGPU, the mental model is:

| Web concept | React Native WebGPU |
| ----------- | --------------------- |
| `<canvas>` in HTML | `<Canvas>` view |
| `canvas.getContext("webgpu")` | `ref.current.getContext("webgpu")` |
| Browser presents frames | You call `context.present()` after submit |

<Callout type="info">
The API is designed to be **symmetric with the web** - same `navigator.gpu`, WGSL pipelines, and synchronous `getContext("webgpu")`. React Native adds manual [`present()`](/docs/rendering/present), native texture import, and a few platform-specific props.
</Callout>

## Basic usage

Mount a `Canvas`, keep a ref, and read the WebGPU context once the view is laid out:

```tsx
import { useEffect, useRef } from "react";
import { StyleSheet, View } from "react-native";
import { Canvas, type CanvasRef } from "react-native-webgpu";

export function Scene() {
  const ref = useRef<CanvasRef>(null);

  useEffect(() => {
    const context = ref.current?.getContext("webgpu");
    if (!context) return;

    // request device, configure context, start render loopâ€¦
  }, []);

  return (
    <View style={{ flex: 1 }}>
      <Canvas ref={ref} style={StyleSheet.absoluteFill} />
    </View>
  );
}
```

`getContext("webgpu")` is **synchronous**, like on the web. If the ref is not ready yet (zero layout size), retry on the next frame with `requestAnimationFrame`.

## Key concepts

| Concept | Description |
| ------- | ----------- |
| **Ref API** | Returns an [`RNCanvasContext`](/api/canvas-ref) with `canvas`, `configure()`, `getCurrentTexture()`, and `present()` |
| **Present** | Call `context.present()` after every `queue.submit()` - see [Present](/docs/rendering/present) |
| **Sizing** | `canvas.clientWidth` / `clientHeight` follow layout; set `canvas.width` / `height` yourself - see [Sizing](/docs/rendering/sizing) |
| **Transparency** | Stack over RN views with the `transparent` prop - see [Transparency](/docs/rendering/transparency) |

## Stacking over React Native UI

A common pattern is a full-screen RN background with the GPU canvas on top:

```tsx
<View style={{ flex: 1 }}>
  <View style={{ flex: 1, backgroundColor: "#3498db" }} />
  <Canvas ref={ref} style={StyleSheet.absoluteFill} transparent />
</View>
```

The GPU layer composites over whatever is beneath it when you configure alpha correctly.

## Hooks

| Hook | Purpose |
| ---- | ------- |
| [`useCanvasRef`](/api/use-canvas-ref) | Ref + optional auto-resize helper |
| [`useDevice`](/api/use-device) | Request adapter/device when the canvas mounts |
| [`useSurface`](/api/use-surface) | Lower-level surface access |

For most apps, `useRef<CanvasRef>` + `useDevice()` is enough.
