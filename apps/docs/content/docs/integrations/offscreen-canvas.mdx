---
title: Offscreen Canvas
description: Headless GPU rendering and CPU readback for testing.
---

Not every GPU workload needs a visible view. [`GPUOffscreenCanvas`](/api/gpu-offscreen-canvas) creates a headless rendering target - useful for automated tests, thumbnail generation, image processing pipelines, and ML preprocessing.

## Basic usage

```tsx
import { GPUOffscreenCanvas } from "react-native-webgpu";

const canvas = new GPUOffscreenCanvas(256, 256);
const context = canvas.getContext("webgpu")!;

const device = await adapter.requestDevice();
context.configure({ device, format: "rgba8unorm" });

// encode render passes into the offscreen target…
device.queue.submit([encoder.finish()]);
// Offscreen contexts do not need present() - no swapchain

const pixels = await canvas.getImageData(); // CPU readback
```

There is no `present()` - nothing is shown on screen. The output is either read back to CPU or used as a texture input to another pass.

## Testing pattern

The package's E2E test suite renders known scenes offscreen and compares pixel output against reference images from Chrome. This validates Dawn behavior without manual QA on every device.

Typical test flow:

1. Create `GPUOffscreenCanvas` at fixed dimensions
2. Render a deterministic scene (solid colors, known geometry)
3. `getImageData()` → compare checksums or per-pixel diff
4. Assert pass/fail

The package E2E suite in `packages/webgpu` uses this pattern.

## When to use offscreen vs Canvas

| | `<Canvas>` | `GPUOffscreenCanvas` |
| --- | ---------- | --------------------- |
| Visible output | Yes | No |
| `present()` | Required | Not used |
| Size | From layout | Fixed at construction |
| Use case | Apps, games, AR | Tests, batch processing |
