---
title: Shared Texture Memory
description: Import IOSurface and AHardwareBuffer surfaces manually.
---

`device.importSharedTextureMemory()` imports a native pixel surface without copying through the CPU.

```tsx
const memory = device.importSharedTextureMemory({
  handle: frame.handle,
  label: "video-frame",
});
const texture = memory.createTexture();

memory.beginAccess(texture, /* initialized */ true);
// sample texture in render pass…
memory.endAccess(texture);

texture.destroy();
frame.release();
```

Use `beginAccess(texture, true)` when the producer has already written pixels (typical for video/camera).

Unlike [External Textures](/docs/native-media/external-textures), you handle YUV conversion yourself in WGSL if the surface is NV12.

<Callout type="info" title="When to use which">
Prefer `importExternalTexture` for camera/video - it handles YUV→RGB on iOS and lifecycle for you. Use shared texture memory when you need direct control over the import/access window.
</Callout>

See [Native Textures](/docs/native-media/native-textures) for the full learning path and [Video Player](/docs/native-media/video-player) for a streaming example.
