title: Video Player
description: Stream decoded MP4 frames into WebGPU with createVideoPlayer.

`RNWebGPU.createVideoPlayer()` decodes a video file and exposes each frame as a [`NativeVideoFrame`](/api/native-types) - the same handle you import with [`importExternalTexture`](/docs/native-media/external-textures) or [`importSharedTextureMemory`](/docs/native-media/shared-texture-memory).

This page walks through streaming decoded MP4 frames into WebGPU with `createVideoPlayer` and `importExternalTexture`.

## When to use it

| Source | API |
| ------ | --- |
| Local / remote MP4 (iOS today) | `createVideoPlayer(url)` |
| Vision Camera live frames | [`createVideoFrameFromNativeBuffer`](/api/rn-webgpu) - see [Vision Camera](/docs/native-media/vision-camera) |
| Tests / Android fallback | `createTestVideoFrame(w, h)` |

Pick the **pixel format** at creation time:

| Format | Import path | WGSL sampler |
| ------ | ----------- | ------------ |
| `"nv12"` (default for external) | `importExternalTexture` | `texture_external` + `textureSampleBaseClampToEdge` |
| `"bgra8"` | `importSharedTextureMemory` | regular `texture_2d` |

```tsx
// NV12 → external texture (recommended for video effects)
const player = RNWebGPU.createVideoPlayer(url, "nv12");

// BGRA → shared texture memory (manual beginAccess/endAccess)
const player = RNWebGPU.createVideoPlayer(url, "bgra8");
```

See [`VideoPlayer`](/api/native-types) and [`CreateVideoPlayerOptions`](/api/native-types) in the API reference.

## 1. Device + canvas

The native-texture feature is enabled automatically when the adapter supports it:

```tsx
const { device } = useDevice();
const ref = useCanvasRef();

const context = ref.current!.getContext("webgpu")!;
context.configure({
  device,
  format: navigator.gpu.getPreferredCanvasFormat(),
  alphaMode: "premultiplied",
});
```

## 2. Create and play

```tsx
const VIDEO_URL =
  "https://example.com/video.mp4";

const player = RNWebGPU.createVideoPlayer(VIDEO_URL, "nv12");
player.play();
```

`copyLatestFrame()` returns the most recently decoded frame, or `null` when the decoder has not produced a new one yet - keep rendering the previous frame to avoid flicker.

## 3. Per-frame render loop

Each animation frame: pull latest pixels, import, draw, submit, present, destroy:

```tsx
let currentFrame: NativeVideoFrame | null = null;

const render = () => {
  const newFrame = player.copyLatestFrame();
  if (newFrame) {
    currentFrame?.release();
    currentFrame = newFrame;
  }

  if (!currentFrame) {
    requestAnimationFrame(render);
    return;
  }

  const externalTex = device.importExternalTexture({
    source: currentFrame,
    label: "video-frame",
  });

  // encode full-screen triangle pass sampling texture_external …
  device.queue.submit([encoder.finish()]);
  context.present();
  externalTex.destroy();

  requestAnimationFrame(render);
};

requestAnimationFrame(render);
```

Order matters: **submit → present → destroy**. Re-import every frame - external textures expire after the submit that sampled them. See [External Textures](/docs/native-media/external-textures).

## 4. Cover-fit UV scaling

Video aspect ratio rarely matches the canvas. Scale UVs around center in a uniform buffer:

```tsx
const computeUvScale = (texW: number, texH: number) => {
  const canvasAR = canvas.width / canvas.height;
  const texAR = texW / texH;
  if (texAR > canvasAR) return [canvasAR / texAR, 1] as const;
  return [1, texAR / canvasAR] as const;
};
```

## 5. Cleanup

```tsx
player.pause();
player.release();
currentFrame?.release();
```

Call `release()` eagerly - waiting for GC stalls the decoder's buffer pool.

## Platform notes

<Callout type="info" title="iOS">
`createVideoPlayer` uses AVPlayer and streams real decoded frames via IOSurface. This is the primary supported platform for hardware video decode.
</Callout>

<Callout type="warn" title="Android / other">
A hardware video decode pipeline is not wired on all platforms yet. Use `createTestVideoFrame(w, h)` as a fallback during development - same render loop, synthetic static frame.
</Callout>
