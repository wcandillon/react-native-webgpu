Compute shaders run **outside** the render pipeline. Instead of drawing triangles, you dispatch a grid of threads that read and write **storage buffers** or **storage textures**. This is how you do particle simulation, image processing, physics, and ML preprocessing on the GPU.

If render shaders answer "what color is this pixel?", compute shaders answer "what is the new state of this data?"

## Typical flow

1. **Create storage buffers** with `GPUBufferUsage.STORAGE` (and often `COPY_DST` for initial upload)
2. **Write WGSL** with an `@compute` entry point and `@workgroup_size(N)`
3. **Build a compute pipeline** from the shader module
4. **Encode a compute pass** - `setPipeline`, `setBindGroup`, `dispatchWorkgroups(x, y, z)`
5. **Use results** in a render pass (same frame) or read back to CPU with `mapAsync`

```tsx
const computePass = encoder.beginComputePass();
computePass.setPipeline(computePipeline);
computePass.setBindGroup(0, bindGroup);
computePass.dispatchWorkgroups(Math.ceil(count / 64));
computePass.end();
```

Each workgroup contains `N` threads (your `@workgroup_size`). `dispatchWorkgroups(4)` with `@workgroup_size(64)` launches 256 threads total.

## Compute vs render

| | Render pass | Compute pass |
| --- | ----------- | -------------- |
| **Purpose** | Draw geometry to attachments | Transform arbitrary data |
| **Shader stages** | Vertex + fragment | Compute only |
| **Output** | Color/depth attachments | Storage buffers/textures |
| **RN present** | Usually followed by `present()` | Often feeds into a render pass first |

On React Native, compute passes do not need `present()` by themselves - only the final render pass that draws to the swapchain does.

## Example use cases

- **Particle simulation** - update positions in compute, draw quads in render ([Particles](/docs/compute/particles))
- **Flocking / boids** - neighbor search in compute, instanced triangles in render ([Compute Boids](/docs/compute/compute-boids))
- **Camera effects** - blur or color grade in compute before compositing ([Vision Camera](/docs/native-media/vision-camera))

## Read/write hazards

A storage buffer cannot be safely read and written in the same compute dispatch. Common patterns:

- **Ping-pong buffers** - read from A, write to B, swap each frame ([Compute Boids](/docs/compute/compute-boids))
- **Separate passes** - compute pass writes, render pass reads (implicit barrier between passes)
