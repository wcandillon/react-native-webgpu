---
title: Render Bundles
description: Pre-record draw commands for static geometry.
---

A **render bundle** records a sequence of draw calls once and replays them cheaply each frame. Static scene geometry (buildings, terrain, asteroids) that does not change can be bundled; dynamic objects (characters, UI) stay outside the bundle.

This reduces CPU encoding overhead - important on mobile where JavaScript time per frame is limited.

## Recording

```tsx
const bundleEncoder = device.createRenderBundleEncoder({
  colorFormats: [presentationFormat],
  depthStencilFormat: "depth24plus",
});

bundleEncoder.setPipeline(staticPipeline);
bundleEncoder.setBindGroup(0, staticBindGroup);
bundleEncoder.setVertexBuffer(0, vertexBuffer);
bundleEncoder.draw(vertices, instanceCount);
// … more static draws

const bundle = bundleEncoder.finish();
```

Record with the same pipeline layout and attachment formats you will use at replay time.

## Replay

```tsx
const pass = encoder.beginRenderPass({ /* attachments */ });
pass.executeBundles([bundle]);

// Dynamic draws after the bundle:
pass.setPipeline(dynamicPipeline);
pass.draw(/* … */);

pass.end();
```

Bundles capture pipeline state and draw commands - not attachment textures. You still create fresh color/depth views each frame from the swapchain.

## When bundles help

| Good fit | Poor fit |
| -------- | -------- |
| Static instanced geometry (thousands of asteroids) | Every object moves every frame |
| UI that rarely changes | Compute-updated vertex data each frame |
| Multi-pass scenes with fixed static pass | Pipelines that change bind groups per frame |

Combine with [Instancing](/docs/compute/instancing) for large static crowds: record one instanced draw in the bundle, replay every frame, add dynamic objects on top.
